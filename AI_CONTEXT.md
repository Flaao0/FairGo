Проект FairGo: Контекст и Правила Разработки
О проекте:
Приложение FairGo — это современный сервис (аналог такси/каршеринга) с картой, геопозицией, профилем
пользователя и оплатой.

Технический стек:

Язык: Kotlin.

UI: 100% Jetpack Compose (никаких XML-разметок для экранов).

Архитектура: MVVM, использование ViewModel, StateFlow для управления состоянием.

Внедрение зависимостей: Dagger Hilt (используется hiltViewModel()).

Навигация: Navigation Compose. Маршруты описаны в sealed class Screen (Welcome, SignIn, SignUp, Map,
AddressSelection, Payment, PromoCode).

Карты: Yandex MapKit SDK 4.x.

Дизайн-система (FairGo Design):

Главный акцентный цвет: Фирменный зеленый Color(0xFF7DB546).

Фоны экранов: Белый Color.White или светло-серый Color(0xFFF2F4F5) / Color(0xFFF7F7F7).

Цвета текста: Темно-серый/синий Color(0xFF3D4754) для заголовков, Color(0xFF6B7A8A) или Color(
0xFF9CA5B4) для плейсхолдеров и подписей.

Скругления (Shapes): Чаще всего используются RoundedCornerShape(16.dp) для кнопок и карточек, 18.dp
для полей ввода.

Кнопки: Главная кнопка (PrimaryGreenButton) — зеленая с белым жирным текстом. Социальные кнопки
круглые, иконки используют tint = Color.Unspecified для сохранения оригинальных цветов SVG.

Текущий статус (Что уже реализовано):

Авторизация: Экраны SignInScreen и SignUpScreen с красивой разметкой текста (AnnotatedString) и
кнопками соцсетей. Запрос разрешений на GPS перенесен на экран приветствия.

Карта (MapScreen): Полностью рабочая карта Yandex.

Настроен UserLocationLayer для слежения за пользователем. При старте камера центрируется с
анимацией.

Логотип Яндекса легально сдвинут в правый нижний угол над нижним баром.

Реализована белая выезжающая шторка "Куда едем?".

Отключен конфликт жестов: свайп меню работает только когда оно открыто (gesturesEnabled =
drawerState.isOpen).

Боковое меню (DrawerMenuContent): Вынесено в отдельный Composable. Содержит аватар, почту и пункты
меню (История, Способ оплаты, Промокод, Поддержка, Выйти).

Способ оплаты (AddCardScreen): Экран ввода банковской карты. Реализована интерактивная
превью-карта (темно-синяя), на которой данные печатаются в реальном времени при вводе. Ограничена
длина полей ввода.

Промокод (PromoCodeScreen): Экран с крупным BasicTextField без рамок. Используется .imePadding(),
чтобы кнопка "Применить" поднималась над клавиатурой.

Строгие правила для ИИ-Агента:

Не удаляй системные отступы: В проекте активно используются systemBarsPadding(),
navigationBarsPadding() и imePadding(). Не стирай их, иначе UI "залезет" под системные элементы
Android.

Yandex MapKit: Помни про специфику SDK 4.x. Логотип скрывать нельзя. Для локации используется слой,
а не флаг isUserLocationLayerEnabled. Центрирование выполняется через
mapView.mapWindow.map.move(...).

Чистый код (DRY): При создании новых элементов старайся выносить их в переиспользуемые приватные
компоненты (как мы сделали с CustomTextField или ActionRow), а не дублировать код.

Навигация: При переходе между экранами используй коллбэки (например, onBackClick: () -> Unit), а
саму логику навигации оставляй в NavGraph.
